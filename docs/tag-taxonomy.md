# 标签规范

## 目标

`tags` 用来做聚类、导航和复用；`keywords` 用来保留文章里的细粒度术语。

如果一个词只对当前文章有意义，或者短期内大概率不会在第二篇文章里复用，就不要放进 `tags`，放到 `keywords`。

## 硬规则

- 普通文章最多 `4` 个标签。
- 系列导读最多 `3` 个标签。
- 每篇文章至少保留 `1` 个能稳定复用的主题标签。
- 优先复用现有标签；新增标签前先检查它是否真的会在后续文章继续使用。
- 正文不要再写首个 `#` 标题，文章页 Hero 已经会渲染标题。

## 禁用标签

以下标签不再使用：

- `我不知道的`
- `系列索引`

这些词属于品牌或内容形态，不提供有效导航价值。

## 规范化替换

- `浏览器原理` -> `浏览器`
- `bug` -> `问题排查`
- `bugs` -> `问题排查`
- `Search` -> `搜索`
- `Performance` -> `性能优化`

## 选择标签的顺序

1. 先放系列 / 主题标签。
2. 再补 1 到 3 个跨文章复用标签。
3. 如果还想加更细的术语，先问自己这词是否会至少在两篇文章里复用。
4. 如果答案不确定，把它写进 `keywords`，不要塞进 `tags`。

## 例子

### 不推荐

```yaml
tags:
  - React
  - ReactDOM
  - 平台抽象
  - 调和
  - 虚拟DOM
  - 我不知道的
```

### 推荐

```yaml
tags:
  - React
  - 调和
  - 虚拟DOM
```

更细的 `ReactDOM`、`平台抽象` 可以放进 `keywords`。
